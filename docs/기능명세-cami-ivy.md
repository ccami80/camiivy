# 기능 명세: 까미 & 아이비 (CAMI & IVY) — 강아지·고양이 프리미엄 커머스 플랫폼

**기능 브랜치**: `cami-ivy-mvp`  
**작성일**: 2025-03-08  
**상태**: 초안  
**입력**: 하나의 플랫폼에 브랜드 둘(까미=강아지, 아이비=고양이). 사용자 구분은 비회원·일반회원·입점업체·관리자. 메인·상품 탐색(카테고리: 의류/산책용품/이동용품/쿠션/계단/식기/위생/놀이/간식/집사와 함께), 상품 상세(AI 상세 템플릿 3종), 장바구니·주문·결제, 마이페이지(반려동물 프로필·주문·리뷰·찜·쿠폰), 입점업체(회원가입·승인·상품 등록·주문·정산), 관리자(승인·큐레이션·통계) 구조 반영.

## Clarifications

### Session 2025-03-08

- Q: 입점업체·관리자 로그인은 일반 로그인과 동일 경로인가? → A: 일반/입점업체는 동일 로그인 경로에서 이메일로 구분 후 역할별 리다이렉트. 관리자는 별도 경로 `/admin/login`.
- Q: 장바구니는 비회원도 사용 가능한가? → A: 비회원도 세션(sessionId) 기준으로 장바구니 사용 가능. 결제 시에는 로그인 유도.
- Q: AI 상세 생성은 상품 등록 시 필수인가? → A: 상품 등록 시 템플릿 3종 중 선택 후 "AI 상세 생성"으로 HTML 생성 가능. 생성 결과를 상세 콘텐츠로 저장하며, 구현 시 해당 플로우 생략 없이 포함.

## 사용자 시나리오 및 검증 *(필수)*

### 사용자 스토리 1 - 메인·상품 탐색 및 상품 상세 조회 (우선순위: P1)

방문자(비회원·회원 공통)가 메인 페이지에서 브랜드(까미/아이비)를 선택하고, 히어로 배너·스토리·추천 카테고리·베스트 상품·리뷰 하이라이트를 본다. 상품 리스트에서는 브랜드·반려동물 종·카테고리·가격 필터와 추천/인기/최신 정렬로 상품을 탐색한다. 상품 상세에서는 이미지·가격·옵션(사이즈/색상)·재고·배송 정보와 AI 생성 상세 콘텐츠(프리미엄 라이프스타일/기능 중심/감성 공감 중 적용된 템플릿), 리뷰·AI 리뷰 요약을 확인할 수 있다.

**우선순위 이유**: 커머스 핵심 가치인 "보기·찾기·확인"이 없으면 구매로 이어질 수 없음.

**독립 검증**: 메인 접속 시 큐레이션 기반 콘텐츠가 노출되고, 상품 목록·상세가 필터·정렬·API 연동으로 정상 표시되면 통과.

**수용 시나리오**:

1. **조건** 메인 페이지 방문, **행동** 브랜드(까미/아이비) 선택, **결과** 해당 브랜드 톤에 맞는 히어로·카테고리·베스트·리뷰 영역이 노출됨
2. **조건** 상품 리스트 페이지, **행동** 브랜드·반려동물·카테고리·가격 필터 및 정렬 선택, **결과** 조건에 맞는 상품만 목록에 표시됨
3. **조건** 상품 상세 페이지, **행동** 상품 선택 후 상세 진입, **결과** 기본 정보·옵션·AI 상세 콘텐츠·리뷰가 표시되고 장바구니 담기·찜·구매하기 가능
4. **조건** 비회원, **행동** 장바구니 담기, **결과** 세션 기준으로 장바구니에 담기며, 주문 시 로그인 유도됨

---

### 사용자 스토리 2 - 회원가입·로그인 및 마이페이지(반려동물·주문·리뷰·찜·쿠폰) (우선순위: P1)

고객이 이메일 회원가입(약관 동의)을 하고 로그인한다. 로그인 후 마이페이지에서 회원 정보 수정, 반려동물 프로필(이름·종·체형·생일 등) 등록·수정·삭제, 주문 내역·주문 상세·취소/환불 신청, 작성한 리뷰 관리, 찜 목록·장바구니 담기, 쿠폰·포인트 조회를 할 수 있다. 비회원은 주문번호·연락처로 주문 조회만 가능하다.

**우선순위 이유**: 구매·리뷰·재구매 등 모든 회원 경험의 기반이 됨.

**독립 검증**: 회원가입·로그인 후 마이페이지 각 메뉴에서 CRUD·조회가 오류 없이 동작하고, 비회원 주문 조회가 가능하면 통과.

**수용 시나리오**:

1. **조건** 비회원, **행동** 회원가입(이메일·비밀번호·이름·전화·약관 동의) 후 저장, **결과** 가입 완료 후 로그인 페이지 또는 자동 로그인
2. **조건** 가입 회원, **행동** 로그인(이메일·비밀번호), **결과** 마이페이지(/my)로 이동, 프로필·반려동물·주문·리뷰·찜·쿠폰 메뉴 접근 가능
3. **조건** 마이페이지, **행동** 반려동물 추가(이름·종·체형 등) 후 저장, **결과** 반려동물 목록에 반영되어 이후 조회·수정·삭제 가능
4. **조건** 주문 완료 회원, **행동** 주문 내역에서 상세 진입 후 취소/환불 신청, **결과** 신청이 반영되고 상태 조회 가능
5. **조건** 비회원, **행동** 비회원 주문 조회 페이지에서 주문번호·연락처 입력, **결과** 해당 주문/배송 정보만 조회됨

---

### 사용자 스토리 3 - 장바구니·주문서·결제 (우선순위: P1)

회원이 장바구니에 담은 상품을 확인하고, 수량·옵션 변경 또는 삭제 후 주문하기를 선택한다. 주문서에서 배송지 선택/입력, 쿠폰 선택, 결제 수단 선택 후 주문을 생성하고 결제를 진행한다. 주문 후 마이페이지에서 주문 내역·상세·배송 조회 및 취소/환불 신청을 할 수 있다.

**우선순위 이유**: 실제 매출이 발생하는 구매 완료 플로우이므로 필수.

**독립 검증**: 장바구니 → 주문서 → 결제(또는 테스트 결제)까지 한 번에 진행 가능하고, 주문·취소/환불이 DB에 반영되면 통과.

**수용 시나리오**:

1. **조건** 회원, 장바구니에 상품 존재, **행동** 주문하기 클릭, **결과** 주문서 페이지로 이동, 배송지·쿠폰·결제 수단 선택 가능
2. **조건** 주문서 작성 완료, **행동** 결제 진행, **결과** 주문 생성·결제 완료 후 주문 완료 화면 및 마이페이지 주문 내역에 반영
3. **조건** 결제 전 주문, **행동** 주문 취소 요청, **결과** 주문 상태가 취소로 변경되고 목록에서 확인 가능
4. **조건** 배송 완료 등 적절한 상태, **행동** 환불 신청, **결과** 환불 요청이 저장되고 관리자/시스템에서 처리 가능

---

### 사용자 스토리 4 - 입점업체 회원가입·승인·상품 등록·주문·정산 (우선순위: P2)

입점 희망자가 입점업체 전용 회원가입(이메일·비밀번호·사업자명·사업자번호·담당자·연락처 등)을 한다. 가입 후 승인 전에는 "승인 대기" 페이지만 접근 가능하다. 관리자가 승인하면 입점업체는 대시보드·상품 목록·상품 등록(기본 정보 + AI 상세 템플릿 선택 후 생성)·상품 수정·주문 관리·정산 조회를 할 수 있다.

**우선순위 이유**: 플랫폼의 입점 구조와 상품 공급·정산이 이루어져야 커머스가 성립함.

**독립 검증**: 입점업체 가입 → 승인 대기 → 관리자 승인 → 상품 등록(AI 상세 포함)·주문 조회·정산 조회가 가능하면 통과.

**수용 시나리오**:

1. **조건** 비회원, **행동** 입점업체 회원가입 폼 작성 후 제출, **결과** 가입 완료 후 로그인 시 승인 대기 상태로 /partner/pending만 접근 가능
2. **조건** 관리자, **행동** 입점업체 목록에서 해당 업체 승인, **결과** 해당 입점업체는 대시보드·상품·주문·정산 메뉴 접근 가능
3. **조건** 승인된 입점업체, **행동** 상품 등록(기본 정보 + 템플릿 선택 후 "AI 상세 생성"), **결과** AI 생성 상세가 반영된 상품이 저장되고 승인 대기 상태로 등록됨
4. **조건** 승인된 입점업체, **행동** 주문 관리·정산 관리 메뉴 진입, **결과** 본인 상품 주문 목록·기간별 정산 목록·상세 조회 가능

---

### 사용자 스토리 5 - 관리자 승인·큐레이션·통계 (우선순위: P2)

관리자는 전용 로그인(/admin/login) 후 대시보드에서 회원·입점업체·상품·주문 통계와 승인 대기 요약을 본다. 입점업체 승인/반려, 상품 승인/반려, 회원 관리·정지, 주문/환불 처리, 메인 큐레이션(히어로·카테고리·베스트·리뷰 하이라이트 등 블록·순서·노출 상품 설정)을 수행할 수 있다.

**우선순위 이유**: 플랫폼 운영·품질 관리·메인 노출 제어에 필수.

**독립 검증**: 관리자 로그인 후 대시보드·입점/상품 승인·주문 처리·큐레이션 CRUD가 동작하면 통과.

**수용 시나리오**:

1. **조건** 관리자, **행동** /admin/login으로 로그인, **결과** 관리자 대시보드로 이동, 승인 대기·통계 확인 가능
2. **조건** 관리자, **행동** 입점업체 목록에서 승인/반려, **결과** 해당 입점업체 상태 변경, 승인 시 해당 업체가 정상 입점 메뉴 이용 가능
3. **조건** 관리자, **행동** 상품 목록·상세에서 승인/반려, **결과** 상품 노출 여부가 승인 상태에 따라 결정됨
4. **조건** 관리자, **행동** 메인 큐레이션에서 블록 추가·순서 변경·노출 상품/이미지 설정·활성화, **결과** 메인 페이지에 반영되어 해당 큐레이션이 노출됨

---

### 예외 상황

- 입점업체 미승인 시: `/partner/pending`만 접근 가능, 대시보드·상품 등록·주문·정산 접근 차단.
- 상품 미승인 시: 고객용 상품 목록·상세에 노출되지 않음. 입점업체는 본인 상품 조회·수정 가능.
- 비회원이 결제 시도 시: 로그인 유도 후 결제 플로우로 복귀.
- 장바구니: 회원은 userId, 비회원은 sessionId 기준으로 유지. 로그인 시 비회원 장바구니 병합 정책은 선택 사항으로 둘 수 있음.

## 요구사항 *(필수)*

### 기능 요구사항

- **FR-001**: 시스템은 하나의 플랫폼에서 브랜드 둘(까미=강아지, 아이비=고양이)을 구분하여 메인·상품 탐색·필터·정렬 시 브랜드·반려동물 종을 반영해야 한다.
- **FR-002**: 시스템은 비회원·일반회원·입점업체·관리자 역할을 구분하며, 일반/입점업체는 동일 로그인 경로에서 이메일로 구분 후 역할별 리다이렉트를 제공하고, 관리자는 별도 로그인 경로를 제공해야 한다.
- **FR-003**: 시스템은 메인 페이지에서 큐레이션(히어로 배너·카테고리·베스트 상품·리뷰 하이라이트)을 관리자 설정에 따라 노출해야 한다.
- **FR-004**: 시스템은 상품 목록 조회 시 브랜드·반려동물 종·카테고리·가격 필터와 추천/인기/최신 정렬을 지원해야 한다.
- **FR-005**: 시스템은 상품 상세에서 기본 정보(이름·가격·옵션·재고·배송), AI 생성 상세 콘텐츠(템플릿 3종: 프리미엄 라이프스타일/기능 중심/감성 공감), 리뷰·AI 리뷰 요약을 표시해야 한다.
- **FR-006**: 시스템은 장바구니를 회원(userId) 또는 비회원(sessionId) 기준으로 유지하고, 담기·수량/옵션 변경·삭제를 지원해야 한다. 결제 시도 시 비회원은 로그인 유도해야 한다.
- **FR-007**: 시스템은 일반회원 가입(이메일·비밀번호·이름·전화·약관 동의), 입점업체 가입(사업자 정보 포함), 관리자 로그인(전용 경로)을 제공해야 한다.
- **FR-008**: 시스템은 일반회원에 대해 마이페이지(프로필·반려동물·주문·리뷰·찜·쿠폰/포인트)와 주문서·결제·취소/환불 신청을 제공해야 한다.
- **FR-009**: 시스템은 비회원 주문 조회(주문번호·연락처)를 제공해야 한다.
- **FR-010**: 시스템은 입점업체 가입 후 승인 전에는 "승인 대기" 페이지만 접근 가능하게 하고, 관리자 승인 후 대시보드·상품 등록/수정(AI 상세 포함)·주문 관리·정산 조회를 제공해야 한다.
- **FR-011**: 시스템은 상품 등록/수정 시 AI 상세 생성(템플릿 선택 후 API 호출로 HTML 생성) 플로우를 포함해야 한다.
- **FR-012**: 시스템은 관리자에게 입점업체·상품 승인/반려, 회원 관리, 주문/환불 처리, 메인 큐레이션 CRUD, 대시보드 통계를 제공해야 한다.

### 주요 엔티티

- **일반회원(User)**: 이메일·비밀번호·이름·전화. 장바구니·주문·반려동물·주소·리뷰·찜·쿠폰·포인트와 연관.
- **입점업체(Partner)**: 이메일·비밀번호·사업자명·사업자번호·담당자·연락처·승인 상태(PENDING/APPROVED/REJECTED). 상품·주문(본인 상품)·정산과 연관.
- **관리자(Admin)**: 이메일·비밀번호·이름. 승인·큐레이션·통계 관리.
- **상품(Product)**: 입점업체 소속, 브랜드·반려동물 종·카테고리·가격·재고·상세 콘텐츠(AI 생성)·승인 상태. 이미지·옵션·variant·리뷰·찜·큐레이션 아이템과 연관.
- **주문(Order)**: 회원 또는 비회원(주문번호·연락처 조회). 주문 항목·배송지·금액·상태·취소/환불과 연관.
- **반려동물(Pet)**: 회원 소속, 이름·종·체형·생일 등. 리뷰 체형 표시 등에 활용.
- **큐레이션(Curation)**: 타입(히어로·카테고리·베스트·리뷰 하이라이트), 브랜드·순서·노출 내용(JSON). 메인 노출 제어.

## 가정

- **범위**: MVP는 웹 커머스·입점·관리자 기능. 앱·라이브 커머스는 제외.
- **결제**: 실제 PG 연동 또는 테스트 결제로 주문 생성·상태 변경까지 구현. 인증 방식( JWT 등)은 구현 방식 문서 참고.
- **AI 상세**: 상품 등록/수정 시 템플릿 선택 후 AI API 호출로 상세 HTML 생성·저장. 외부 AI API 연동 여부는 구현 단계에서 결정 가능.
- **UI/UX**: 롯데온(정보 구조 명확)·텐바이텐(감성 큐레이션) 참고, 과한 귀여움 지양, 세련된 프리미엄 톤 유지.
- **데이터**: 모든 페이지·API는 mock이 아닌 실제 DB(Prisma·MySQL) 연동 필수. 관리자/입점업체/페이지 추가 시 해당 API 동시 구현.

## 성공 기준 *(필수)*

### 측정 가능한 결과

- **SC-001**: 비회원·회원이 메인 접속 후 1분 이내에 브랜드 선택·큐레이션·상품 목록 진입이 가능하다.
- **SC-002**: 회원이 로그인 후 마이페이지의 프로필·반려동물·주문·리뷰·찜·쿠폰 메뉴에 오류 없이 접근하고, CRUD가 DB에 반영된다.
- **SC-003**: 회원이 장바구니 → 주문서 → 결제(또는 테스트 결제)까지 한 세션 내에서 완료할 수 있다.
- **SC-004**: 입점업체가 승인 후 상품 등록(AI 상세 생성 포함)·수정·주문 조회·정산 조회를 할 수 있고, 관리자가 해당 입점업체·상품 승인/반려를 수행할 수 있다.
- **SC-005**: 관리자가 메인 큐레이션을 수정한 뒤 메인 페이지에 변경 내용이 반영된다.
- **SC-006**: 모든 페이지가 권한(비회원/회원/입점업체/관리자)에 따라 접근 허용 또는 차단되며, mock이 아닌 실제 DB 연동으로 동작한다.
