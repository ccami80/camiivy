1차로 해야 할일

이 프로젝트는 강아지·고양이 용품을 판매하는
프리미엄 반려동물 이커머스 플랫폼이다.

브랜드 구조:
- 까미 (CAMI): 강아지 용품 전용
- 아이비 (IVY): 고양이 용품 전용

하나의 플랫폼 안에서 브랜드만 분리된 구조다.

==================================================

[기술 스택 - 필수 고정]

- Frontend: Next.js (App Router)
- Language: JavaScript (TypeScript 사용 금지)
- Styling: Tailwind CSS
- Backend: Next.js API Routes
- Database: MySQL
- ORM: Prisma (JavaScript 기준)
- Authentication: JWT 기반 인증

※ TypeScript, tRPC, GraphQL 등은 사용하지 않는다.
※ JavaScript 기준으로만 코드 작성한다.

==================================================

[서비스 방향 & 톤]

- 전반적으로 고급스럽고 미니멀한 분위기
- 귀엽기만 한 반려동물 쇼핑몰은 지양
- 사람이 봐도 세련된 커머스 UI
- 여백, 타이포그래피, 정돈된 레이아웃 중심

==================================================

[사용자 권한 구조]

1. 비회원
2. 일반회원 (구매자)
3. 입점업체 (셀러)
4. 관리자

각 권한은 접근 가능한 페이지와 기능이 명확히 분리된다.
권한 체크 없는 페이지는 허용하지 않는다.

==================================================

[페이지 구조]

[Public]
- 메인 페이지
- 로그인 / 회원가입
- 상품 리스트
- 상품 상세
- 장바구니

[User - 마이페이지]
- 회원 정보 관리
- 반려동물 프로필 관리
- 주문 내역
- 리뷰 관리
- 찜 목록
- 쿠폰 / 포인트

[Partner - 입점업체]
- 입점업체 회원가입
- 입점업체 승인 대기 상태
- 상품 등록 / 수정 / 삭제
- 주문 관리
- 정산 관리

[Admin - 관리자]
- 관리자 로그인
- 관리자 대시보드
- 회원 관리
- 입점업체 승인 / 반려
- 상품 승인 / 반려
- 주문 / 환불 / 정산 관리
- 메인 큐레이션 관리

==================================================

[상품 구조 - 필수]

- 상품명
- 브랜드 (CAMI / IVY)
- 반려동물 타입 (강아지 / 고양이)
- 카테고리
- 가격
- 재고
- 옵션 (사이즈, 색상 등)
- 배송 정보
- 상세 콘텐츠

==================================================

[상품 상세 페이지 - AI 자동 생성 기능]

상품 상세 페이지에는 반드시 아래 기능이 포함된다.

- 관리자 또는 입점업체가 입력 폼 작성
- 입력 데이터를 기반으로 AI가 상세 콘텐츠 자동 생성
- 고급스러운 상세 페이지 템플릿 3종 제공

템플릿 유형:
1. 프리미엄 라이프스타일형
2. 기능 중심 설득형
3. 감성 공감형

템플릿 선택 후
→ 상세 페이지 콘텐츠가 자동 완성되는 구조

※ 이 기능은 필수이며 생략하거나 단순화하지 않는다.

==================================================

[관리자 / 입점업체 폼 설계 원칙]

- 모든 관리 페이지는 실제 동작하는 폼으로 구현
- mock UI, placeholder 전용 화면 금지
- 모든 데이터는 Prisma ORM과 연결
- CRUD 전체 흐름 구현

==================================================

[개발 규칙]

- 제공된 PRD를 100% 기준으로 개발
- 임의 기능 생략, 축소, 변경 금지
- 페이지 단위로 구현
- 더미 데이터라도 전체 플로우는 동작해야 한다

==================================================

[1단계 요청]

지금은 코드를 작성하지 말고,
전체 프로젝트 폴더 구조와
페이지 구조만 설계해줘.

각 페이지의 역할을 간단히 설명해줘.








2차로 해야 할일


[벤치마크 사이트]

UI/UX 및 구조 참고:
- 롯데온: 대형 커머스의 안정적인 정보 구조, 신뢰감 있는 레이아웃
- 텐바이텐: 감성적인 큐레이션, 상품 분류 방식, 과하지 않은 위트

요청 사항:
- 롯데온처럼 복잡하지 않되, 정보 구조는 명확하게
- 텐바이텐처럼 가볍지만 유치하지 않은 감성
- 반려동물 쇼핑몰 특유의 과한 귀여움은 지양
- “사람이 들어도 세련돼 보이는” 화면 구성

위 벤치마크를 참고하되, 디자인을 그대로 복제하지 말고
프리미엄 반려동물 브랜드에 맞게 재해석한다.
