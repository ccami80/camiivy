## 3. 페이지별 역할 요약

### 3-1. Public (비회원·일반회원 공통)

| 경로 | 페이지 | 역할 |
|------|--------|------|
| `/` | 메인 | 브랜드(까미/아이비) 선택, 히어로, 스토리, 추천 카테고리·베스트·리뷰 하이라이트, 관리자 큐레이션 노출 |
| `/(public)/products` | 상품 리스트 | 브랜드·반려동물·카테고리·가격 필터, 정렬(추천/인기/최신), 목록 표시 |
| `/(public)/products/[id]` | 상품 상세 | 기본 정보(명/가격/옵션/재고/배송), AI 생성 상세 템플릿 3종, 리뷰·AI 리뷰 요약 |
| `/(public)/cart` | 장바구니 | 담긴 상품 목록, 수량·옵션 변경, 삭제, 주문하기 이동 (비회원은 로그인 유도) |

---

### 3-2. Auth (비로그인 전용)

| 경로 | 페이지 | 역할 |
|------|--------|------|
| `/(auth)/login` | 로그인 | 이메일/비밀번호, 소셜 로그인, 일반/입점업체 구분 없이 로그인 후 역할별 리다이렉트 |
| `/(auth)/signup` | 회원가입 | 일반회원(구매자) 이메일 회원가입, 약관 동의 |
| `/(auth)/order-inquiry` | 비회원 주문 조회 | 주문번호·연락처로 주문/배송 조회 |

---

### 3-3. User — 마이페이지 (일반회원 전용)

| 경로 | 페이지 | 역할 |
|------|--------|------|
| `/(user)/my` | 마이페이지 홈 | 주문 요약, 찜/쿠폰/포인트 요약, 바로가기 |
| `/(user)/my/profile` | 회원 정보 관리 | 프로필 수정, 비밀번호 변경 |
| `/(user)/my/pets` | 반려동물 프로필 | 반려동물 등록·수정·삭제 (종, 체형 등) |
| `/(user)/my/orders` | 주문 내역 | 주문 목록, 상태별 필터 |
| `/(user)/my/orders/[id]` | 주문 상세 | 주문 내용, 배송 조회, 취소/환불 신청 |
| `/(user)/my/reviews` | 리뷰 관리 | 작성한 리뷰 목록, 수정·삭제 |
| `/(user)/my/wishlist` | 찜 목록 | 찜한 상품 목록, 삭제, 장바구니 담기 |
| `/(user)/my/coupons` | 쿠폰/포인트 | 보유 쿠폰, 사용 내역, 포인트 잔액·내역 |

주문서·결제는 `/(public)` 또는 `/(user)` 하위 단일 경로(예: `/checkout`)로 두고, 비회원이면 로그인 유도, 회원이면 결제 진행하도록 권한만 분기하는 방식 권장.

---

### 3-4. Partner — 입점업체

| 경로 | 페이지 | 역할 |
|------|--------|------|
| `/(partner)/partner/signup` | 입점업체 회원가입 | 셀러 전용 가입 폼, 사업자 정보 등 |
| `/(partner)/partner/pending` | 승인 대기 | 승인 대기 시 노출, 대시/상품 등록 접근 차단 |
| `/(partner)/partner` | 입점업체 대시보드 | 주문·매출·상품 현황 요약 |
| `/(partner)/partner/products` | 상품 목록 | 내 상품 목록, 승인 상태, 수정/삭제 진입 |
| `/(partner)/partner/products/new` | 상품 등록 | 상품 기본 정보 입력 + AI 상세 생성(템플릿 3종 선택 후 자동 생성) |
| `/(partner)/partner/products/[id]/edit` | 상품 수정 | 기존 상품 수정, AI 상세 재생성 가능 |
| `/(partner)/partner/orders` | 주문 관리 | 입점업체 주문 목록, 배송/처리 상태 변경 |
| `/(partner)/partner/settlement` | 정산 관리 | 정산 주기별 매출·정산 금액·내역 조회 |

---

### 3-5. Admin — 관리자

| 경로 | 페이지 | 역할 |
|------|--------|------|
| `/(admin)/admin/login` | 관리자 로그인 | 관리자 전용 로그인 (일반/입점업체와 분리) |
| `/(admin)/admin` | 관리자 대시보드 | 회원·주문·입점·상품 통계, 최근 주문/승인 대기 요약 |
| `/(admin)/admin/users` | 회원 관리 | 일반회원 목록·검색·상세·정지 등 |
| `/(admin)/admin/partners` | 입점업체 관리 | 입점 신청 목록, 승인/반려, 입점업체 목록 관리 |
| `/(admin)/admin/products` | 상품 관리 | 전체 상품 목록, 승인/반려, 검색·필터 |
| `/(admin)/admin/products/[id]` | 상품 상세(관리자) | 상품 정보 확인, 승인/반려 처리 |
| `/(admin)/admin/orders` | 주문/환불/정산 | 주문 목록, 환불 처리, 정산 관련 조회 |
| `/(admin)/admin/curation` | 메인 큐레이션 | 메인 노출 상품·배너·순서 설정 (CRUD) |

## 4. 권한별 접근 정리

| 구분 | 비회원 | 일반회원 | 입점업체 | 관리자 |
|------|--------|----------|----------|--------|
| 메인, 상품리스트, 상품상세, 장바구니 | ✅ | ✅ | ✅ | ✅ |
| 로그인/회원가입/비회원 주문조회 | ✅ | 리다이렉트 | 리다이렉트 | - |
| 마이페이지 전역 | ❌ | ✅ | ❌ | ❌ |
| 입점업체 전역 | ❌ | ❌ | ✅(승인 후) | ❌ |
| 관리자 전역 | ❌ | ❌ | ❌ | ✅ |
| 주문/결제 | 장바구니까지 가능, 결제 시 로그인 유도 | ✅ | - | - |

- 모든 페이지는 **권한 체크 후** 접근 허용/거부 처리.
- 입점업체 미승인 시: `/(partner)/partner/pending`만 허용, 나머지 입점업체 라우트 차단.

## 5. 상품 상세 — AI 자동 생성 기능 (구조만 명시)

- **입력**: 관리자 또는 입점업체가 상품 기본 정보 + 상세용 입력 폼 작성.
- **처리**: API `api/ai-detail` 등에서 입력 기반 AI 상세 콘텐츠 생성.
- **템플릿**: 1) 프리미엄 라이프스타일형, 2) 기능 중심 설득형, 3) 감성 공감형 — 선택 시 해당 템플릿으로 상세 콘텐츠 자동 완성.
- **위치**: 상품 등록/수정 페이지(`/(partner)/partner/products/new`, `.../products/[id]/edit`)에 템플릿 선택 + “AI 상세 생성” 플로우 포함. 생략·단순화 없이 구현 대상.

## 6. UI/UX 방향 (참고)

- 벤치마크: 롯데온(정보 구조·신뢰감) + 텐바이텐(감성 큐레이션·가벼운 위트).
- 요청: 복잡하지 않되 정보 구조 명확, 가볍되 유치하지 않음, 과한 귀여움 지양, “사람이 봐도 세련된” 구성.
- 상세: **prd/04-UI-UX-벤치마크-및-디자인-원칙.md** 참고.

## 7. 공통 컴포넌트·유틸 (참고)

- **components/layout**: `Header`, `Footer`, `BrandSwitch(CAMI/IVY)`, `MainNav`, `UserNav`, `PartnerNav`, `AdminSidebar`.
- **components/product**: `ProductCard`, `ProductOption`, `ProductDetailTemplate`(3종), `ReviewList`, `ReviewSummary`.
- **lib/auth**: JWT 발급/검증, 역할(role) 판별, 미들웨어 또는 라우트 단위 권한 체크.
- **contexts**: `AuthContext`(user, partner, admin 구분), `CartContext` 등.
